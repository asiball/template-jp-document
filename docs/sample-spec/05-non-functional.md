# 非機能要件

本 API は、次に示す非機能要件を満たすものとする。

- **可用性**: 月間稼働率 99.9% 以上を維持すること。計画メンテナンスは事前告知のうえ深夜帯に実施する。
- **性能**: `GET /items/{sku}` の応答時間は、95 パーセンタイルで 200 ミリ秒以内とすること。
- **拡張性**: ロケーション数の増加に対して線形に近いスケールアウトが可能な設計とすること。
- **監査性**: すべての在庫変動操作は、操作者・操作日時・変更前後の値を監査ログへ記録すること。
- **セキュリティ**: すべての通信は TLS 1.2 以上で暗号化し、アクセストークンの有効期限は最長 1 時間とすること。

これらの非機能要件は四半期ごとにレビューを行い、実測値との乖離が大きい項目については改善計画を別途策定するものとする。特に性能要件については、繁忙期（月末月初）のトラフィック増加を考慮した負荷試験を年に一度実施し、結果を本書の改訂履歴に反映すること。
