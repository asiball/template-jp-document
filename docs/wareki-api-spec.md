# はじめに

## 目的

本書は、和暦・西暦の相互変換を行う Python ライブラリ `wareki`（以下、「本ライブラリ」という）が提供する外部公開関数の仕様を定めることを目的とする。本ライブラリを利用するアプリケーション開発者が、関数の引数・戻り値・送出しうる例外を正確に把握し、意図した挙動でライブラリを組み込めるようにすることを目指す。

## 適用範囲

本書が対象とする範囲は、`wareki` パッケージ直下からインポート可能な公開関数・公開クラス・公開例外に限定される。アンダースコアで始まるモジュール（`wareki._internal` 等）や、元号テーブルの内部データ構造、改元日の判定アルゴリズムの実装詳細は本書の対象外であり、これらはバージョン間で予告なく変更されうる非公開実装として扱う。

## 用語定義

本書で用いる主要な用語を次の表に示す。

| 用語 | 説明 |
|------------------|------------------------------------------------------------------------|
| 元号 | 日本の紀年法で用いられる年号。本ライブラリは明治以降の元号を対象とする。 |
| 改元日 | ある元号から次の元号へ切り替わる日。改元日当日は新元号の元年に帰属する。 |
| グレゴリオ暦 | 現行の太陽暦。本ライブラリが西暦日付として扱う暦法。 |
| 先発グレゴリオ暦 | グレゴリオ暦（英: proleptic Gregorian calendar）を、それが実際に採用される以前の日付にまで遡って適用したと仮定する暦法。Python の `datetime.date` はこの方式で日付を表現しており、本ライブラリもこれに準拠する。 |
| 元年 | ある元号における最初の年。和暦年としては `1` を指定する（`0` は指定不可）。 |

: 用語定義一覧

# ライブラリ概要

本ライブラリは、西暦(`datetime.date`)と和暦（元号 + 年月日）の間の変換、および和暦表記の文字列の解析を提供する。対応する元号は明治・大正・昭和・平成・令和の 5 元号であり、各元号の在位期間および改元日は付録「対応元号一覧」に定める値を内部テーブルとして保持する。

## 対応環境

本ライブラリの対応環境を次の表に示す。

| 項目 | 内容 |
|------------------|------------------------------------------------------------------------|
| Python | 3.10 以上 |
| OS | Linux、macOS、Windows（プラットフォーム依存処理を含まない） |
| 依存パッケージ | なし（Python 標準ライブラリのみで実装） |
| タイムゾーン | `datetime.date` のみを扱い、タイムゾーン情報は扱わない |

: 対応環境一覧

## インポート方法

本ライブラリは次のいずれかの方法でインポートする。

```python
import wareki

wareki.to_wareki(...)
```

```python
from wareki import to_wareki, to_gregorian, parse, era_of, eras
```

## スレッド安全性・イミュータブル方針

本ライブラリの公開関数はすべて純粋関数として実装されており、モジュールレベルの可変状態を持たない。元号テーブルはモジュールのインポート時に一度だけ構築され、以後変更されることはない。したがって、複数スレッドから公開関数を同時に呼び出しても安全である。

`era_of` および `eras` が返す `Era` データクラスは `frozen=True` で定義されたイミュータブルなオブジェクトであり、生成後にフィールドを変更することはできない。呼び出し側が `Era` インスタンスをキャッシュして再利用する場合も、値が意図せず書き換わる心配はない。

# 共通仕様

## 例外体系

本ライブラリが送出する例外はすべて `WarekiError` を基底クラスとする。呼び出し側は個別の例外クラスを捕捉することも、`WarekiError` で一括して捕捉することもできる。

| 例外クラス | 継承元 | 説明 |
|------------------------|------------------------|--------------------------------------------------------|
| `WarekiError` | `Exception` | 本ライブラリが送出する例外の基底クラス。個別の派生クラスを区別せず捕捉したい場合に使用する。 |
| `EraNotFoundError` | `WarekiError` | 指定した元号名・元号コード（例: `"令和"`、`"R"`）が既知の元号に一致しない場合に送出される。 |
| `DateOutOfRangeError` | `WarekiError` | 日付が本ライブラリの対応範囲（1873-01-01 以降）の外、または和暦の年月日が該当元号の在位期間の外にある場合に送出される。 |
| `AmbiguousDateError` | `WarekiError` | 和暦文字列から元号・年を一意に確定できない場合に送出される（例: 元号名を伴わない `"元年5月1日"`）。 |

: `wareki` 例外クラス一覧

## 日付の受け入れ型

西暦日付を引数に取る公開関数は、次のいずれかの型を受け付ける。

- `datetime.date`（`datetime.datetime` のインスタンスは受け付けない。日時から日付への暗黙変換によるタイムゾーン起因の誤変換を避けるため、`TypeError` を送出する）
- ISO 8601 形式の文字列(`"YYYY-MM-DD"`)。この形式に一致しない文字列を渡した場合は `ValueError` を送出する。

## バージョニング方針

本ライブラリは [Semantic Versioning](https://semver.org/) に従う。公開 API とは、`wareki` パッケージ直下から到達可能な名前、すなわち「公開関数仕様」章に列挙する公開関数群、`Era` データクラス、および `WarekiError` とその派生例外クラスを指す。

アンダースコアで始まるモジュール・関数・属性（`wareki._internal.era_table` 等）はいかなる意味でも公開 API ではなく、パッチバージョンの更新であっても予告なく変更・削除されることがある。公開 API の破壊的変更はメジャーバージョンの更新時にのみ行う。

# 公開関数仕様

本章では、`wareki` パッケージが提供する公開関数を関数ごとに説明する。

## to_wareki

```python
def to_wareki(
    date: datetime.date | str,
    *,
    format: str = "long",
) -> str: ...
```

西暦日付を和暦表記の文字列に変換する。`format` 引数によって出力形式を切り替えることができる。

引数を次の表に示す。

| 引数名 | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `date` | `datetime.date` / `str` | 必須 | 変換対象の西暦日付。文字列の場合は ISO 8601 形式(`"YYYY-MM-DD"`)。 |
| `format` | `str` | 任意 | 出力形式。`"long"`（例: `"令和8年7月14日"`。ただし改元初年は `"令和元年5月1日"` のように「元年」と表記する）または `"short"`（例: `"R8.7.14"`。`"long"` と異なり `"short"` 形式では改元初年も数字の `1` で `"R1.5.1"` のように表記する）。省略時は `"long"`。 |

: `to_wareki` の引数一覧

戻り値: `str` — 指定した形式で表現された和暦文字列。

送出する例外を次の表に示す。

| 例外 | 発生条件 |
|---------------------------|---------------------------------------------------------------|
| `DateOutOfRangeError` | `date` が本ライブラリの対応範囲（1873-01-01 以降）より前の日付である場合。 |
| `ValueError` | `format` に `"long"` `"short"` 以外の値を指定した場合、または `date` が ISO 8601 形式の文字列として不正な場合。 |
| `TypeError` | `date` に `datetime.datetime` など未対応の型を渡した場合。 |

: `to_wareki` が送出する例外

使用例を次に示す。

```python
>>> import datetime
>>> from wareki import to_wareki
>>> to_wareki(datetime.date(2026, 7, 14))
'令和8年7月14日'
>>> to_wareki("2026-07-14", format="short")
'R8.7.14'
>>> to_wareki(datetime.date(2019, 5, 1))
'令和元年5月1日'
>>> to_wareki(datetime.date(2019, 5, 1), format="short")
'R1.5.1'
```

> **注意:** 改元日当日はすべて新元号の元年に帰属する。したがって `2019-05-01`（平成から令和への改元日）は `'平成31年5月1日'` ではなく `'令和元年5月1日'` に変換される（`"long"` 形式では改元初年を「元年」と表記するため。`"short"` 形式では `'R1.5.1'` のように数字の `1` で表記する）[^accession-day]。

## to_gregorian

```python
def to_gregorian(
    era: str,
    year: int,
    month: int,
    day: int,
) -> datetime.date: ...
```

和暦の元号・年・月・日から `datetime.date` を生成する。

引数を次の表に示す。

| 引数名 | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `era` | `str` | 必須 | 元号。漢字表記（例: `"令和"`）またはローマ字 1 文字の元号コード（例: `"R"`）のいずれかを受け付ける。 |
| `year` | `int` | 必須 | 和暦年。元年は `1` を指定する（`0` は指定不可）。 |
| `month` | `int` | 必須 | 月(`1`–`12`)。 |
| `day` | `int` | 必須 | 日。 |

: `to_gregorian` の引数一覧

戻り値: `datetime.date` — 対応する西暦日付。

送出する例外を次の表に示す。

| 例外 | 発生条件 |
|---------------------------|---------------------------------------------------------------|
| `EraNotFoundError` | `era` に指定した元号名・元号コードが既知のどの元号にも一致しない場合。 |
| `DateOutOfRangeError` | 指定した `year`/`month`/`day` が該当元号の在位期間の範囲外の場合、または `year` に `0` 以下の値を指定した場合。 |
| `ValueError` | `month`/`day` の組み合わせがグレゴリオ暦として存在しない日付の場合（例: 2月30日）。 |

: `to_gregorian` が送出する例外

使用例を次に示す。

```python
>>> from wareki import to_gregorian
>>> to_gregorian("令和", 8, 7, 14)
datetime.date(2026, 7, 14)
>>> to_gregorian("R", 1, 5, 1)
datetime.date(2019, 5, 1)
```

> **注意:** 現行の元号（本書執筆時点では令和）には終了日が定められていないため、`year` に上限は設けていない。将来の改元後も、旧元号の在位期間内であればそのまま変換できる。

## parse

```python
def parse(text: str) -> datetime.date: ...
```

和暦表記の文字列を解析し、`datetime.date` を返す。`"令和8年7月14日"` のような長い形式、および `"R8.7.14"` のような短縮形式の双方に対応する。

引数を次の表に示す。

| 引数名 | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `text` | `str` | 必須 | 解析対象の和暦文字列。 |

: `parse` の引数一覧

戻り値: `datetime.date` — 解析結果の西暦日付。

送出する例外を次の表に示す。

| 例外 | 発生条件 |
|---------------------------|---------------------------------------------------------------|
| `AmbiguousDateError` | 元号名を伴わない `"元年5月1日"` のような文字列など、どの元号を指しているか一意に確定できない場合。 |
| `EraNotFoundError` | 文字列中の元号名・元号コードが既知のどの元号にも一致しない場合。 |
| `DateOutOfRangeError` | 解析結果の年月日が該当元号の在位期間の範囲外の場合。 |
| `ValueError` | 文字列が本関数の対応する既知の書式（長い形式・短縮形式のいずれ）にも一致しない場合。 |

: `parse` が送出する例外

使用例を次に示す。

```python
>>> from wareki import parse
>>> parse("令和8年7月14日")
datetime.date(2026, 7, 14)
>>> parse("R8.7.14")
datetime.date(2026, 7, 14)
```

> **注意:** `"元年"` という表記は元号名と併記された場合（例: `"令和元年5月1日"`）にのみ解析可能である。元号名を省略した `"元年5月1日"` は `AmbiguousDateError` となる。

## era_of

```python
def era_of(date: datetime.date | str) -> Era: ...
```

指定した日付が属する元号を取得する。戻り値は元号の情報を保持する `Era` データクラスのインスタンスである。

引数を次の表に示す。

| 引数名 | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `date` | `datetime.date` / `str` | 必須 | 対象の西暦日付。文字列の場合は ISO 8601 形式。 |

: `era_of` の引数一覧

戻り値: `Era` — 次のフィールドを持つイミュータブルなデータクラス。

- `name: str` — 元号の漢字表記（例: `"令和"`）
- `code: str` — 元号のローマ字 1 文字コード（例: `"R"`）
- `start: datetime.date` — 当該元号の開始日（改元日）
- `end: datetime.date | None` — 当該元号の終了日。現行の元号の場合は `None`。

送出する例外を次の表に示す。

| 例外 | 発生条件 |
|---------------------------|---------------------------------------------------------------|
| `DateOutOfRangeError` | `date` が本ライブラリの対応範囲（1873-01-01 以降）より前の日付の場合。 |

: `era_of` が送出する例外

使用例を次に示す。

```python
>>> import datetime
>>> from wareki import era_of
>>> era = era_of(datetime.date(2019, 5, 1))
>>> era.name
'令和'
>>> era.start
datetime.date(2019, 5, 1)
>>> era.end is None
True
```

## eras

```python
def eras(*, since: datetime.date | str | None = None) -> list[Era]: ...
```

本ライブラリが対応する元号の一覧を取得する。

引数を次の表に示す。

| 引数名 | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `since` | `datetime.date` / `str` / `None` | 任意 | 指定した場合、この日付以降に開始した元号のみを返す。省略時(`None`)は明治以降の全元号を返す。 |

: `eras` の引数一覧

戻り値: `list[Era]` — 開始日の昇順に並んだ `Era` のリスト。

送出する例外を次の表に示す。

| 例外 | 発生条件 |
|---------------------------|---------------------------------------------------------------|
| （なし） | 本関数は通常、例外を送出しない。`since` に未来の日付を指定した場合は、単に空のリストを返す（エラーとはしない）。 |

: `eras` が送出する例外

使用例を次に示す。

```python
>>> from wareki import eras
>>> [e.name for e in eras()]
['明治', '大正', '昭和', '平成', '令和']
>>> import datetime
>>> [e.name for e in eras(since=datetime.date(1990, 1, 1))]
['平成', '令和']
```

# 非推奨API・互換性

過去のバージョンで公開していた一部の関数・属性は、より一貫性のある名前を持つ関数に置き換えられ、非推奨として維持されている。非推奨 API を次の表に示す。

| 非推奨API | 非推奨バージョン | 代替 | 削除予定バージョン |
|------------------------|--------------------|------------------------------------|--------------------|
| `to_wareki_str(date)` | 0.6 | `to_wareki(date)` | 1.0.0 |
| `Era.label` | 0.9 | `Era.name`（`Era.label` と同じく属性であり、呼び出し方法ではなく参照方法のみが変わる） | 1.2.0 |
| `strict_parse(text)` | 0.8 | `parse(text)` | 1.0.0 |

: 非推奨API一覧

非推奨 API は呼び出し時に `DeprecationWarning` を送出するが、削除予定バージョンに到達するまでは従来どおり動作し続ける。

## 後方互換性の約束

本ライブラリは次の後方互換性の方針を約束する。

- マイナーバージョン・パッチバージョンの更新では、公開 API の既存の挙動（引数の意味、戻り値の型、送出する例外の種類）を破壊しない。
- ある公開 API を非推奨とする場合、少なくとも 2 回のマイナーバージョン更新を経てから削除する。
- 元号テーブルへの新元号の追加（将来の改元への対応）はマイナーバージョンの更新として扱い、既存の元号に関する挙動には影響しない[^new-era-addition]。

# 付録: 対応元号一覧 {.unnumbered}

本ライブラリが対応する元号の一覧と在位期間を次の表に示す。時代区分によって元号をグループ化しており、この列の結合は Markdown 標準の表記法では表現できないため、Pandoc の生 Typst 記法（`` ```{=typst} `` フェンス）を用いて `table.cell(rowspan: ...)` によるセル結合を行っている。なお、時代区分は本表の便宜上の区分であり、`wareki` ライブラリの API や内部データ構造が「時代区分」という概念を持つわけではない。

> **注意:** 明治 6 年（1873 年）1 月 1 日のグレゴリオ暦採用より前は、日本国内では太陰太陽暦が用いられていた。本ライブラリはこの期間の日付を先発グレゴリオ暦へ一意に対応付けることをせず、`DateOutOfRangeError` の対象としている。したがって、和暦上は明治元年から続く「明治」であっても、本ライブラリが実際に変換を保証するのは 1873-01-01 以降の日付に限られる。

```{=typst}
#figure(
  table(
    columns: (15%, 15%, 55%, 15%),
    align: (center + horizon, center + horizon, center + horizon, center + horizon),
    table.header([時代区分], [元号], [期間（改元日基準）], [元号コード]),
    table.hline(),
    table.cell(rowspan: 3)[近代],
    [明治], [1868-01-25 〜 1912-07-29], [M],
    [大正], [1912-07-30 〜 1926-12-24], [T],
    [昭和], [1926-12-25 〜 1989-01-07], [S],
    table.hline(),
    table.cell(rowspan: 2)[現代],
    [平成], [1989-01-08 〜 2019-04-30], [H],
    [令和], [2019-05-01 〜 （現在）], [R],
  ),
  caption: [対応元号一覧],
  kind: table,
)
```

なお、`eras()` が返すリストの `start` フィールドは上表の「期間」列の開始日と一致するが、本ライブラリが実際に日付変換を保証する範囲は前述のとおり 1873-01-01 以降である点に留意すること。

[^accession-day]: 改元日当日の帰属をどちらの元号とするかは実務上しばしば混乱の原因となるため、本ライブラリでは「改元日当日は新元号の元年」というルールを明文化し、`to_wareki` と `parse` の双方で一貫して適用している。
[^new-era-addition]: 実運用では、皇室典範上の改元は政令公布から施行までの期間が短いことが多い（令和への改元では 2019 年 4 月 1 日の元号発表から施行の 2019 年 5 月 1 日まで 1 か月であった）。本ライブラリの元号テーブルは事前にハードコードされているため、施行前の新元号を先行して扱うことはできない。
