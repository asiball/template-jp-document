# API仕様

本章では、本 API が提供する主要なエンドポイントについて説明する。ベース URL は `https://api.example.internal/inventory/v1` とする。

## エンドポイント一覧

| メソッド | パス | 概要 |
|-----------|----------------------------------|---------------------------------------------|
| GET | `/items/{sku}` | 指定した SKU の在庫情報を取得する |
| GET | `/items` | 在庫一覧を検索条件付きで取得する |
| POST | `/items/{sku}/movements` | 入出庫イベントを登録する |
| POST | `/items/{sku}/reservations` | 在庫引当を作成する |
| DELETE | `/items/{sku}/reservations/{id}` | 在庫引当を解除する |
| POST | `/stocktakes` | 棚卸結果を登録する |

: API エンドポイント一覧

## 共通エラー仕様

本 API がエラーを返却する場合、HTTP ステータスコードに加えて、レスポンスボディの `error.code` フィールドにエラーコードを設定する。クライアントは個々のメッセージ文字列ではなく、この `error.code` の値によってエラー種別を判定すること。エラーコードは、対象リソースや状態を表す英単語をアンダースコアで連結したスネークケース（例: `SKU_NOT_FOUND`）で命名する。

本 API が返却する主なエラーコードを次の表に示す。以降の各節では、原則としてこの表に定義したエラーコードのみを参照する。

| エラーコード | HTTPステータス | 説明 |
|------------------------------|--------------------|--------------------------------------------------|
| `VALIDATION_ERROR` | 400 | リクエストパラメータが不正である（必須項目の欠落、型不一致、値域外など）。 |
| `SKU_NOT_FOUND` | 404 | 指定した SKU が存在しない。 |
| `RESERVATION_NOT_FOUND` | 404 | 指定した引当 ID が存在しない、または既に解除済みである。 |
| `INSUFFICIENT_STOCK` | 409 | 引当可能数量が引当要求数量に満たない。 |
| `STOCKTAKE_CONFLICT` | 409 | 同一 SKU・ロケーションに対する棚卸登録が短時間に重複した。 |
| `INTERNAL_ERROR` | 500 | サーバ内部でエラーが発生した。 |

: 共通エラーコード一覧

## 在庫情報の取得

`GET /items/{sku}` は、指定した SKU の在庫情報をロケーション単位で返却する。レスポンスの例を次に示す。

```json
{
  "sku": "SKU-00012345",
  "name": "段ボール箱(A4サイズ)",
  "locations": [
    { "location": "WH1-A-001", "available": 120, "reserved": 15 },
    { "location": "WH2-B-014", "available": 40, "reserved": 0 }
  ],
  "updatedAt": "2026-07-14T09:30:00+09:00"
}
```

curl での呼び出し例は次のとおりである。

```bash
curl -sS \
  -H "Authorization: Bearer ${ACCESS_TOKEN}" \
  "https://api.example.internal/inventory/v1/items/SKU-00012345"
```

レスポンスのステータスコードが `200` 以外の場合、クライアントは「共通エラー仕様」に示す `error.code` フィールド（例: `SKU_NOT_FOUND`）を参照してエラー種別を判定すること。

SKU の指定には `sku` パスパラメータを用いる。`sku` は `SKU-` に続く 8 桁の数字で構成される 12 文字の識別子である（例: `SKU-00012345`）。

## 在庫一覧の検索

`GET /items` は、検索条件に合致する在庫情報の一覧をページング付きで返却する。クエリパラメータを次に示す。

| パラメータ | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `q` | string | 任意 | SKU または商品名の部分一致検索文字列（省略時は絞り込みなし）。 |
| `location` | string | 任意 | 絞り込み対象のロケーション（省略時は全ロケーション）。 |
| `limit` | integer | 任意 | 取得件数の上限（1〜100）。省略時は 20。 |
| `offset` | integer | 任意 | 取得開始位置（0 起算）。省略時は 0。 |

: 在庫一覧検索のクエリパラメータ

レスポンスの例を次に示す。

```json
{
  "items": [
    {
      "sku": "SKU-00012345",
      "name": "段ボール箱(A4サイズ)",
      "totalAvailable": 160
    }
  ],
  "total": 1,
  "limit": 20,
  "offset": 0
}
```

`limit` に 100 を超える値、または `offset` に負の値を指定した場合、`VALIDATION_ERROR` を返却する。

## 入出庫の登録

`POST /items/{sku}/movements` は、指定した SKU の入庫または出庫イベントを記録し、在庫数量へ即時反映する。FR-002 が定める入出庫イベントの記録は、このエンドポイントによって実現される。

リクエストボディの主なフィールドを次に示す。

| フィールド | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `type` | string | 必須 | 移動区分。`"inbound"`（入庫）/`"outbound"`（出庫）。 |
| `quantity` | integer | 必須 | 移動数量。1 以上の整数。 |
| `location` | string | 必須 | 対象ロケーション。 |
| `reason` | string | 任意 | 移動理由コード。省略時は `"unspecified"`。 |

: 入出庫登録リクエストのフィールド

登録が成功した場合、レスポンスは `201 Created` とともに、発行された入出庫イベント ID と更新後の在庫数量を含むボディを返却する。

`type` に `"inbound"`/`"outbound"` 以外の値を指定した場合、または `quantity` に 1 未満の値を指定した場合は `VALIDATION_ERROR` を返却する。`type` が `"outbound"` であり、かつ指定数量が引当可能数量を超える場合は `INSUFFICIENT_STOCK` を返却する。

## 在庫引当の作成

`POST /items/{sku}/reservations` は、指定した SKU に対して在庫引当を作成する。リクエストボディの主なフィールドを次に示す。

| フィールド | 型 | 必須 | 説明 |
|----------------|--------------------|---------|---------------------------------------------|
| `quantity` | integer | 必須 | 引当数量。1 以上の整数。 |
| `orderId` | string | 必須 | 引当元となる受注 ID。 |
| `location` | string | 任意 | 引当対象のロケーション（省略時は自動選択）。 |

: 引当作成リクエストのフィールド

引当が成功した場合、レスポンスは `201 Created` とともに発行された引当 ID を含むボディを返却する。この引当 ID は、`DELETE /items/{sku}/reservations/{id}` による引当解除リクエストで指定する識別子として用いる。引当可能数量が要求数量に満たない場合は `INSUFFICIENT_STOCK` を返却する（処理フローの詳細は「機能要件」章を参照）。

## 在庫引当の解除

`DELETE /items/{sku}/reservations/{id}` は、`id` で指定した在庫引当を解除し、引当済み数量を引当可能数量へ戻す(FR-003)。出荷キャンセル時に呼び出すことを想定する。

パスパラメータを次に示す。

| パラメータ | 説明 |
|----------------|--------------------------------------------------------------|
| `sku` | 引当対象の SKU。 |
| `id` | 解除する引当の引当 ID。引当作成（「在庫引当の作成」節）のレスポンスで発行された値。 |

: 引当解除のパスパラメータ

解除に成功した場合、レスポンスは `204 No Content` を返却しボディを持たない。指定した引当 ID が存在しない、または既に解除済みの場合は `RESERVATION_NOT_FOUND` を返却する。

## 棚卸結果の登録

`POST /stocktakes` は、実地棚卸で確認した実在庫数を登録し、帳簿上の在庫数量との差異（棚卸差異）を記録する(FR-004)。

リクエストボディの主なフィールドを次に示す。

| フィールド | 型 | 必須 | 説明 |
|--------------------|--------------------|---------|---------------------------------------------|
| `sku` | string | 必須 | 棚卸対象の SKU。 |
| `location` | string | 必須 | 棚卸対象のロケーション。 |
| `countedQuantity` | integer | 必須 | 実地棚卸で確認した実在庫数。0 以上の整数。 |
| `reasonCode` | string | 任意 | 差異理由コード（差異が 0 でない場合は必須）。 |

: 棚卸登録リクエストのフィールド

登録が成功した場合、レスポンスは `201 Created` とともに、記録された棚卸差異の値を含むボディを返却する。同一 SKU・ロケーションへの棚卸登録が短時間に重複した場合は `STOCKTAKE_CONFLICT` を、差異があるにもかかわらず `reasonCode` を指定しなかった場合は `VALIDATION_ERROR` を返却する。
